## 用户需求

用户提出三个优化需求：

1. **页面自动刷新**：跳转到审批中心、主页时自动刷新数据，用户无需手动刷新
2. **工单中止通知优化**：

- 当工单因下一步骤未找到审批人而中止时，消息中心推送消息给申请人
- 消息内容："您的xxx申请意外中止，请点击查看"
- 审批备注写明："因下一步骤未找到审批人，系统自动中止申请"
- 审批历史最后一步的操作人显示"系统"
- 点击通知跳转到审批中心

3. **驳回通知增强**：推送消息包含驳回人信息，格式："您的xxx申请已被xxx驳回，请点击查看"，点击后跳转审批中心

## 产品概述

优化工作流通知机制和页面刷新体验，提升用户感知和操作便捷性。

## 核心功能

- 页面自动刷新（审批中心、主页）
- 工单中止时的系统通知和日志记录
- 驳回通知包含驳回人信息

## 实现方案

### 1. 页面自动刷新

**审批中心**：

- 当前：onShow 只在 `currentList.length === 0` 时刷新
- 修改：移除条件判断，每次 onShow 都刷新数据

**主页**：

- 已实现：onShow 中调用 syncUserProfile、syncNotifications、loadAnnouncements

### 2. 工单中止通知优化

**当前流程**：

- handleApproval 返回 warningMessage → 前端弹窗 → 用户手动确认中止

**优化后流程**：

- handleApproval 检测未找到审批人 → 自动中止工单 → 发送通知 → 记录日志（操作人为"系统"）

**实现细节**：

- 新增 `sendOrderTerminatedNotification` 函数发送中止通知
- 中止日志的 operatorId 设为 'system'，operatorName 设为 '系统'
- 工单的 reviewRemark 字段记录中止原因

### 3. 驳回通知增强

修改 `sendTaskCompletedNotification` 函数，当 approvalResult 为 'rejected' 时：

- content 格式改为："您的{orderTypeName}申请已被{approverName}驳回，请点击查看"
- 需要传入驳回人信息

```javascript
// 发送工单中止通知
async function sendOrderTerminatedNotification(order, reason) {
  const orderTypeName = getOrderTypeName(order.orderType)
  await sendAppNotification(order.businessData.applicantId, {
    type: 'order_terminated',
    title: '申请已中止',
    content: `您的${orderTypeName}申请意外中止，请点击查看`,
    orderId: order._id,
    orderNo: order.orderNo,
    orderType: order.orderType,
    reason: reason
  })
}
```

### workflowEngine/index.js - 驳回通知增强

```javascript
// 修改 sendTaskCompletedNotification 函数签名，增加 approverName 参数
async function sendTaskCompletedNotification(order, approvalResult, comment, approverName) {
  const orderTypeName = getOrderTypeName(order.orderType)
  let content = ''
  
  if (approvalResult === 'rejected') {
    content = `您的${orderTypeName}申请已被${approverName || '审批人'}驳回，请点击查看`
  } else {
    content = comment || '您的申请已通过审批'
  }
  
  await sendAppNotification(order.businessData.applicantId, {
    type: 'task_completed',
    title: approvalResult === 'approved' ? '审批通过' : '审批驳回',
    content: content,
    orderId: order._id,
    // ...
  })
}
```

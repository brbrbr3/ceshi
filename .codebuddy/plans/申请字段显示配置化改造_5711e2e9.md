## 用户需求

申请卡片和申请详情页显示的字段目前是硬编码的，需要改为从数据库动态读取。

## 问题分析

1. **`approval.js` 的 `mapRequestItem` 函数**：根据 `orderType` 硬编码返回不同字段（第 93-139 行）
2. **`approval-detail.wxml`**：根据 `orderType` 硬编码显示不同字段（第 15-96 行）
3. 每新增申请类型需修改多处代码，维护成本高

## 解决方案

扩展 `workflow_templates` 表添加 `displayConfig` 字段，定义每种申请类型的卡片和详情页显示配置。创建工单时快照到 `workflowSnapshot`，前端根据配置动态渲染。

## 技术方案

### 1. 数据库设计

扩展 `workflow_templates` 表，添加 `displayConfig` 字段：

```
{
  "displayConfig": {
    "cardFields": [
      {"field": "patientName", "label": "就医人"},
      {"field": "relation", "label": "关系"}
    ],
    "detailFields": [
      {"field": "patientName", "label": "就医人姓名"},
      {"field": "relation", "label": "与申请人关系"},
      {"field": "institution", "label": "就医机构"},
      {"field": "otherInstitution", "label": "机构名称", "condition": {"field": "institution", "value": "其他"}}
    ]
  }
}
```

### 2. 字段配置说明

| 属性 | 说明 |
| --- | --- |
| `field` | 字段名（对应 businessData 中的字段） |
| `label` | 显示标签 |
| `condition` | 可选，条件显示配置 |


### 3. 修改文件清单

| 文件 | 修改内容 |
| --- | --- |
| `workflow_templates` 表 | 添加 `displayConfig` 字段到现有模板 |
| `cloudfunctions/workflowEngine/index.js` | 创建工单时快照 `displayConfig` |
| `cloudfunctions/officeAuth/index.js` | 返回 `displayConfig` 字段 |
| `miniprogram/pages/office/approval/approval.js` | `mapRequestItem` 改为动态生成 |
| `miniprogram/components/approval-detail/approval-detail.wxml` | 改为动态渲染字段 |
| `miniprogram/components/approval-detail/approval-detail.js` | 添加动态字段处理逻辑 |


### 4. 数据流程

```
workflow_templates.displayConfig
    ↓ 创建工单时快照
work_orders.workflowSnapshot.displayConfig
    ↓ 云函数返回
前端 request.displayConfig
    ↓ 动态渲染
卡片/详情页
```

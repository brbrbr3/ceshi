## 用户需求

### 1. 个人外出报备功能

- 在 home 页常用功能添加"外出报备"图标入口
- 页面顶部显示当前外出状态卡片（有未返回时显示）
- 动态按钮：外出报备 / 返回报备
- 点击"外出报备"：从下方弹出卡片式窗口填写信息（目的地、同行人、返回时间、出行方式）
- 点击"返回报备"：弹出确认对话框
- 按钮下方标题"出行记录"，列表按月分组显示，分页加载
- 列表显示：日期、出发时间-返回时间、目的地、同行人、出行方式

### 2. 出行管理 Dashboard

- 在 home 页常用功能添加"出行管理"图标入口
- 仅馆领导、部门负责人、管理员可见
- 包含统计图表（本月出行次数、按部门/出行方式统计）

### 3. 超时提醒

- 用户超过计划返回时间1小时，推送消息提醒

### 4. 编码规范要求

- 常量在 initSystemConfig 中定义，前端通过 constants.js 获取
- 使用 READONLY 安全规则
- 使用分页加载框架（pagination behavior）
- UI 参照标准页面（渐变色头部、卡片式布局）

## 技术方案

### 数据库设计

**集合名**：`trip_reports`

**字段结构**：

```javascript
{
  _id: String,              // 记录ID
  _openid: String,          // 创建者openid（READONLY规则检查）
  userName: String,         // 用户姓名
  department: String,       // 所属部门
  destination: String,      // 目的地
  companions: String,       // 同行人
  plannedReturnAt: Number,  // 计划返回时间戳
  travelMode: String,       // 出行方式
  departAt: Number,         // 外出时间戳
  returnAt: Number,         // 实际返回时间戳（null表示未返回）
  status: String,           // 状态：'out' | 'returned' | 'overtime'
  overtimeNotified: Boolean, // 是否已发送超时通知
  createdAt: Number,        // 创建时间戳
  updatedAt: Number         // 更新时间戳
}
```

**安全规则**：`READONLY`（用户可创建，所有人可读）

**索引**：

- `idx_openid_status`：openid + status 组合索引
- `idx_departAt`：departAt 降序索引

### 关键实现细节

1. **弹窗表单**：使用微信小程序内置 `action-sheet` 组件或自定义底部弹出层
2. **分页加载**：使用 `behaviors/pagination.js`
3. **按月分组**：前端根据 departAt 字段计算月份，分组显示
4. **权限控制**：Dashboard 入口根据用户角色动态显示
5. **图表**：使用 wx-charts 或自定义 Canvas 绘制简单柱状图/饼图

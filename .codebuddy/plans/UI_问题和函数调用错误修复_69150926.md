## 用户需求

1. **标题换行**：通知公告页和每周菜单页的条目标题太长时应允许换行显示
2. **返回刷新**：从通知详情页返回通知公告页后，页面应自动刷新
3. **picker箭头位置**：发布通知页面的通知类型picker，箭头">"应在控件最右端
4. **修复错误**：提交菜单时报错 `utils.showtoast is not a function`

## 问题定位

| 问题 | 文件 | 原因 |
| --- | --- | --- |
| 标题不换行 | `announcement-list.wxss` 第185-187行 | `.announcement-item-title` 设置了 `white-space: nowrap` |
| 标题不换行 | `menus.wxss` 第123-124行 | `.menu-item-title` 同样设置了 `white-space: nowrap` |
| 返回不刷新 | `announcement-list.js` 第25-31行 | `onShow` 只在列表为空时加载，未检测返回操作 |
| picker箭头位置 | `announcement-create.wxss` 第13-17行 | 箭头紧跟内容，未固定在右端 |
| showToast错误 | `menu-edit.js` 多处 | 调用 `utils.showtoast`（小写t），但 `utils.js` 导出的是 `showToast`（大写T） |


## 技术方案

### 1. 标题换行修改

移除 `white-space: nowrap`，添加 `word-wrap: break-word` 和 `line-clamp` 限制最大行数。

### 2. 返回刷新实现

在 `onShow` 中添加 `this.resetPagination()` 和 `this.loadAnnouncements()` 实现刷新。

### 3. picker箭头位置修复

修改 `.auth-picker-content` 为 `justify-content: space-between`，让箭头固定在右端。

### 4. showToast错误修复

将 `menu-edit.js` 中所有 `utils.showtoast` 改为 `utils.showToast`（共6处）。

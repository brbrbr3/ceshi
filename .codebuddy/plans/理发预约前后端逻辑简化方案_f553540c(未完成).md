## 产品概述

重构理发预约功能的前后端交互逻辑，大幅简化后端职责，将业务逻辑判断移至前端。

## 核心功能

1. **前端负责**：计算显示日期、判断14:20截止时间、组合展示数据
2. **后端负责**：仅查询数据库返回各日期的时段预约情况
3. **节假日查询**：复用现有 `holidayManager` 云函数
4. **减少请求次数**：一次请求获取所有日期的时段预约情况，不再单独调用 `getAvailableSlots`

## 技术栈

- 微信小程序原生开发
- CloudBase 云函数
- NoSQL 文档数据库

## 实现方案

### 数据流重构

**重构前（多次请求）：**

```
前端计算日期 → 调用 getDisplayDates → 返回日期信息
选择日期 → 调用 getAvailableSlots → 返回时段列表
```

**重构后（一次请求）：**

```
前端计算日期 → 调用 holidayManager 获取节假日
            → 调用 getReservationSlots 获取各日期时段预约情况
            → 前端组合数据渲染
```

### 后端接口设计

**新增接口：`getReservationSlots`**

- 输入：`dates: ['2026-03-30', '2026-04-01', '2026-04-03']`
- 输出：

```
{
  "slotsByDate": {
    "2026-03-30": ["14:30", "15:30"],  // 已预约的时段
    "2026-04-01": [],
    "2026-04-03": ["16:00", "17:00", "17:30"]
  }
}
```

**复用接口：`holidayManager.getByYear`**

- 输入：`{ action: 'getByYear', params: { year: 2026 } }`
- 输出：

```
{
  "code": 0,
  "data": {
    "config": { "year": 2026, "dates": ["2026-04-01", "2026-05-01"] }
  }
}
```

### 前端数据处理流程

```
1. calculateDisplayDates() → 本地计算周一三五日期
2. 调用 holidayManager.getByYear → 获取节假日集合
3. 调用 haircutManager.getReservationSlots → 获取各日期已预约时段
4. 本地组合数据：
   - 判断 isToday（本地时间）
   - 判断 isFullyDisabled（本地时间14:20后）
   - 判断 isHoliday（节假日集合）
   - 计算 reservationCount（已预约时段数量）
   - 生成时段列表 slots（7个时段 × 是否已预约）
```

## 实现细节

### 后端修改要点

1. **删除 `getDisplayDates` 接口** - 前端不再需要
2. **删除 `getAvailableSlots` 接口** - 合并到新接口
3. **新增 `getReservationSlots` 接口**：

- 批量查询多个日期的预约记录
- 只返回 `{ date: timeSlot }` 格式的数据
- 使用 `_.in()` 批量查询，减少数据库请求次数

4. **删除冗余函数**：

- `isDayFullyDisabled()` - 前端本地判断
- `getHolidays()` - 使用 holidayManager 替代
- 日期格式化相关函数 - 前端自行处理

### 前端修改要点

1. **合并加载流程**：

- `loadDisplayDates()` 改为并行调用节假日和预约数据
- 删除 `loadSlots()` 方法

2. **本地数据组合**：

```javascript
buildDateDisplayData(dates, holidays, slotsByDate) {
// 组合日期、节假日、预约情况
// 判断 isToday, isFullyDisabled, isHoliday
// 生成完整的 displayDates 数组
}
```

3. **时段列表生成**：

```javascript
buildSlotsList(date, bookedSlots) {
// 根据 TIME_SLOTS 配置生成时段列表
// 标记 isBooked 状态
}
```

### 性能优化

1. **减少云函数调用**：从 2-4 次减少到 2 次（节假日 + 预约数据）
2. **批量数据库查询**：使用 `_.in()` 一次性查询所有日期
3. **前端缓存**：节假日数据可按年份缓存

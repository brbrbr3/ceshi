## 用户需求

审批中心申请卡片和申请详情页的进度条不显示。

## 问题分析

经过代码调试，发现根本原因：

1. 云函数 `getApprovalData` 返回的数据中**缺少 `workflowSnapshot` 字段**
2. 前端 `mapRequestItem` 尝试从 `request.workflowSnapshot` 获取步骤信息
3. 由于 `workflowSnapshot` 是 `undefined`，导致 `totalSteps = 0`
4. 进度条条件 `totalSteps >= 2` 永远不满足

## 数据库工单结构

工单数据包含 `workflowSnapshot` 和 `currentStep` 字段，但云函数返回时未传递。

## 修改目标

修改 `cloudfunctions/officeAuth/index.js`，在以下位置添加 `workflowSnapshot` 和 `currentStep` 字段：

- mineList 构建逻辑（第 709-802 行）
- pendingList 构建逻辑（第 889-972 行）
- doneList 构建逻辑（第 1075-1152 行）

## 技术方案

### 修改文件

| 文件 | 修改内容 |
| --- | --- |
| `cloudfunctions/officeAuth/index.js` | 在 mineList、pendingList、doneList 构建逻辑中添加 `workflowSnapshot` 和 `currentStep` 字段 |


### 修改详情

#### 1. mineList（第 709-802 行）

在每个订单类型的返回对象中添加：

```javascript
workflowSnapshot: order.workflowSnapshot,
currentStep: order.currentStep
```

#### 2. pendingList（第 889-972 行）

在每个订单类型的返回对象中添加：

```javascript
workflowSnapshot: order.workflowSnapshot,
currentStep: order.currentStep
```

#### 3. doneList（第 1075-1152 行）

在每个订单类型的返回对象中添加：

```javascript
workflowSnapshot: order.workflowSnapshot,
currentStep: order.currentStep
```

### 数据流程

```
work_orders 表
    ↓ 包含 workflowSnapshot.steps 和 currentStep
云函数 getApprovalData
    ↓ 需要传递这两个字段
前端 mapRequestItem
    ↓ 从 workflowSnapshot.steps 获取 totalSteps
approval-card / approval-detail 组件
    ↓ 根据 totalSteps >= 2 显示进度条
```

## 需求概述

修复工作流审批中心的管理员权限过滤逻辑：管理员应该只看到自己实际需要处理的任务，而不是看到所有任务。

## 核心问题

当前代码存在硬编码问题，按工单类型和步骤名称判断权限，导致：

1. 管理员能看到所有就医申请任务（不当逻辑）
2. 新增申请类型需要修改代码，可扩展性差
3. 工作流模板调整后代码需同步修改

## 修复目标

- 消除所有硬编码，完全基于工作流模板驱动的权限判断
- 管理员只能看到分配给自己的任务
- 新增申请类型无需修改代码

## 技术方案

### 核心洞察

`workflowEngine` 在创建任务时已根据工作流模板计算好 `approverList`，`officeAuth` 无需重复实现权限判断。

### 任务结构

```javascript
{
  approverType: 'role' | 'user',
  approverId: '角色ID 或 用户openid',
  approverList: [{id, name}, ...],  // 有权限审批的用户列表（由工作流引擎计算）
}
```

### 修复方案

将第 852-896 行的过滤逻辑替换为通用逻辑：

```javascript
pendingList = pendingTasksResult.data.filter(task => {
  // 1. 直接分配的任务（approverType === 'user'）
  if (task.approverId === openid) {
    return true
  }
  
  // 2. 角色审批任务：检查是否在审批人列表中
  if (task.approverType === 'role' && task.approverList) {
    return task.approverList.some(approver => approver.id === openid)
  }
  
  return false
})
```

### 优势

1. **零硬编码**：所有权限判断由工作流模板驱动
2. **可扩展**：新增申请类型无需修改代码
3. **易维护**：修改工作流步骤只需改模板，不改代码
4. **一致性**：与 `workflowEngine` 的权限检查逻辑保持一致

### 影响范围

- 仅影响待审批列表的过滤逻辑
- 不影响已处理列表
- 所有权限由工作流模板配置决定

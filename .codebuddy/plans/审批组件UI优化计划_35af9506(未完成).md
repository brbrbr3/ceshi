## 用户需求

1. **卡片间距**：审批中心的申请卡片之间显示应该留有一定空隙
2. **审批日志颜色**：approval-detail 组件中，审批历史日志条目类型为"审批通过"的，左侧蓝线改为绿色
3. **进度条显示逻辑优化**：进度条不应硬编码仅针对"就医申请"类型显示，应改为基于工作流步骤数判断（步骤数>=2且状态为待审批时显示），提高代码可扩展性

## 修改目标

- `approval-card.wxss`：添加卡片间距样式
- `approval-detail.wxss`：添加审批通过状态的颜色样式
- `approval-card.wxml`：优化进度条显示条件
- `approval-detail.wxml`：优化进度条显示条件、审批日志颜色判断

## 技术方案

### 1. 卡片间距

在 `approval.wxml` 中，卡片列表容器使用 `wx:for` 渲染 `approval-card` 组件。由于组件独立渲染，使用相邻兄弟选择器 `.approval-list-card + .approval-list-card` 无法生效。

**方案**：在父容器 `<view wx:if="{{currentList.length}}">` 添加样式类，使用 `gap` 属性或给每个组件添加 `margin-bottom`。

### 2. 审批日志颜色

工作流日志的 `action` 值：

- `start`：发起申请（蓝色）
- `approve`：审批通过（绿色）
- `reject`：驳回（红色）
- `terminate`：中止（红色）

**方案**：在 WXSS 添加 `.is-approved` 样式类，WXML 中判断 `item.action === 'approve'`。

### 3. 进度条显示逻辑

**当前代码**（硬编码）：

```
wx:if="{{request.orderType === 'medical_application' && request.showProgress && request.progress}}"
```

**优化后**（可扩展）：

```
wx:if="{{request.showProgress && request.progress && request.progress.totalSteps >= 2 && request.status === 'pending'}}"
```

这样未来新增任何类型的多步骤审批，只要 `totalSteps >= 2` 且状态为待审批，就会自动显示进度条。

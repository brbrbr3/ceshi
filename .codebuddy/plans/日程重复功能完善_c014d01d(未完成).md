## 产品概述

完善日历日程的重复功能，使重复日程能够正确显示和管理，并支持基于工作日的重复类型。

## 核心功能

1. **详情页显示重复信息**：在日程详情弹窗中显示重复类型及截止日期
2. **重复日程按规则显示**：时间轴日程列表中，重复日程按设置的重复规则在对应日期显示
3. **必填重复截止日期**：设置重复时必须同时设置截止日期，范围限制为今天至当年最后一天
4. **工作日重复类型**：新增"工作日每天"、"工作日每周"，根据节假日配置判断工作日
5. **整体系列操作**：编辑、删除重复日程时操作整个系列，而非单个实例

## 技术栈

- 云函数：Node.js + wx-server-sdk
- 前端：微信小程序原生框架
- 数据库：CloudBase NoSQL

## 技术方案

### 一、数据库变更

`calendar_schedules` 集合新增 `repeatEndDate` 字段：

- 类型：String（YYYY-MM-DD格式）
- 仅当 `repeat !== 'none'` 时有效
- 约束：>= startDate 且 <= 当年最后一天（如 2026-12-31）

`repeat` 字段扩展：

| 值 | 显示名称 | 说明 |
| --- | --- | --- |
| none | 不重复 | 默认值 |
| daily | 每天 | 每天重复 |
| weekly | 每周 | 每周同星期几重复 |
| monthly | 每月 | 每月同日期重复 |
| workdayDaily | 工作日每天 | 每个工作日重复（排除节假日和周末） |
| workdayWeekly | 工作日每周 | 每周同星期几，仅工作日 |


### 二、工作日判断逻辑

**节假日配置来源**：`holiday_configs` 集合

```javascript
// 工作日判断函数
function isWorkday(dateStr, holidayDates) {
  const date = new Date(dateStr)
  const dayOfWeek = date.getDay() // 0=周日, 6=周六
  
  // 周末非工作日
  if (dayOfWeek === 0 || dayOfWeek === 6) return false
  
  // 节假日非工作日
  if (holidayDates.includes(dateStr)) return false
  
  return true
}
```

### 三、云函数修改

**文件**：`cloudfunctions/scheduleManager/index.js`

1. **handleCreate**：添加 repeatEndDate 参数处理

- 当 repeat !== 'none' 时，repeatEndDate 必填
- 验证 repeatEndDate <= 当年最后一天

2. **handleUpdate**：支持更新 repeatEndDate

3. **handleGetByDate**：核心改动 - 计算重复实例

- 获取当年节假日配置（缓存）
- 查询非重复日程：startDate <= date <= endDate
- 查询重复日程并计算是否在当天出现：
    - daily：每天出现
    - weekly：每周同星期几出现
    - monthly：每月同日期出现
    - workdayDaily：每天出现，但需判断是否为工作日
    - workdayWeekly：每周同星期几出现，但需判断是否为工作日
- 合并返回

4. **handleGetScheduleDates**：计算重复日期标记

- 获取当年节假日配置
- 遍历重复日程，根据规则生成所有出现日期
- workdayDaily/workdayWeekly 类型需过滤非工作日

## 实现说明

- 重复实例计算在云函数端执行，前端无需额外处理
- 截止日期选择器使用 datetime-picker 组件，限制日期范围
- 时区处理遵循项目现有规范（使用 utils.parseLocalDate）
- 工作日判断依赖 `holiday_configs` 集合，需确保节假日配置完整
- 当年最后一天动态计算：`new Date(currentYear, 11, 31)`

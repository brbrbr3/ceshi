## 产品概述

创建"本馆日历"页面，展示日历视图并支持管理员/会计配置节假日。配置的节假日将在日历中以"休"字角标显示。

## 核心功能

1. **日历展示**

- 使用 wx-calendar 组件展示月视图
- 支持农历显示（使用 wc-plugin-lunar 插件）
- 已配置的节假日显示"休"角标

2. **节假日配置**

- 仅管理员和会计可操作
- 弹窗式配置界面
- 年份选择：今年/明年
- 动态添加节假日日期（点击"+"递增）
- 提交后存入数据库并实时更新日历

3. **入口导航**

- home 页日历卡片点击"查看日历"跳转

## 技术栈

- **前端框架**：微信小程序原生开发
- **日历组件**：@lspriv/wx-calendar + @lspriv/wc-plugin-lunar
- **数据库**：CloudBase NoSQL
- **云函数**：Node.js

## 实现方案

### 1. wx-calendar 组件集成

根据组件文档，关键配置：

```javascript
// 属性
view: 'month'           // 月视图
weekstart: 0            // 周日起始
marks: []               // 日期标记（角标）
customNavBar: false     // 使用系统导航栏

// marks 结构
{
  date: '2026-03-21',   // 日期 YYYY-MM-DD
  type: 'rest',         // 标记类型
  text: '休'            // 显示文字
}
```

### 2. 数据库设计

新建集合 `holiday_configs`：

```javascript
{
  _id: String,           // 记录ID
  year: Number,          // 年份（如 2026）
  dates: [String],       // 节假日日期数组 ['2026-01-01', '2026-01-02']
  createdBy: String,     // 创建者openid
  createdByName: String, // 创建者姓名
  createdAt: Number,     // 创建时间戳
  updatedAt: Number      // 更新时间戳
}
```

安全规则：`ADMINWRITE`（所有用户可读，仅管理员可写）

### 3. 云函数设计

新建云函数 `holidayManager`，支持以下 action：

- `getByYear`：获取指定年份节假日配置
- `save`：保存节假日配置（覆盖式更新）

### 4. 权限控制

```javascript
// 判断是否可配置
const canConfig = user.isAdmin || user.role === '会计' || user.role === '会计主管'
```

## 实现要点

### 农历插件启用

```javascript
const { WxCalendar } = require('@lspriv/wx-calendar/lib')
const { LunarPlugin } = require('@lspriv/wc-plugin-lunar')
WxCalendar.use(LunarPlugin)
```

### 动态日期 Picker

- 使用数组存储节假日列表
- 新增时取上一个日期 +1 天作为默认值
- 使用 `wx:` 循环渲染多个 picker

### 弹窗样式

复用 trip-report 的弹窗样式类：

- `.trip-popup-mask`：遮罩层
- `.trip-popup-container`：弹窗容器
- `.trip-popup-header`：弹窗头部
- `.trip-popup-form`：表单区域
- `.trip-submit-btn`：提交按钮

## 交互设计

1. **日历滑动**：左右滑动切换月份
2. **配置弹窗**：从底部滑出，点击遮罩关闭
3. **添加日期**：点击"+"按钮，在列表末尾新增日期选择器
4. **提交反馈**：提交成功后显示 toast 并关闭弹窗

## Agent Extensions

### MCP

- **readNoSqlDatabaseStructure**
- Purpose: 查询现有数据库结构，确认是否需要新建 holiday_configs 集合
- Expected outcome: 获取数据库集合列表，验证新集合需求

- **writeNoSqlDatabaseContent**
- Purpose: 创建 holiday_configs 集合并初始化结构
- Expected outcome: 成功创建集合并设置安全规则

- **createFunction**
- Purpose: 创建 holidayManager 云函数
- Expected outcome: 云函数创建成功，可处理节假日配置的读写操作

### SubAgent

- **code-explorer**
- Purpose: 搜索项目中权限判断的现有实现，确保权限逻辑一致
- Expected outcome: 找到权限判断的通用模式，复用到日历配置权限

## 产品概述

为外出报备功能增加同行人自动设置外出状态的能力

## 核心功能

1. **同行人识别**：用户输入同行人姓名（多个用空格分隔），系统检测是否为已注册的小程序用户
2. **自动设置外出状态**：报备成功后，自动将识别到的同行人设置为"外出中"状态
3. **独立返回报备**：返回报备时仅更新当前用户状态，同行人需自行报备返回
4. **状态反馈**：提交时告知用户哪些同行人被成功设置为外出状态

## 技术方案

### 1. 数据库修改

为 `office_users` 表添加 `isOut` 字段：

- 字段名：`isOut`
- 类型：Boolean
- 默认值：false
- 说明：true = 外出中，false = 在岗

### 2. 云函数修改 (`cloudfunctions/tripReport/index.js`)

#### handleDepart 修改

```javascript
// 1. 解析同行人姓名（空格分隔）
const companionNames = companions ? companions.split(/\s+/).filter(Boolean) : []

// 2. 按姓名批量查询用户
const matchedUsers = await usersCollection
  .where({ name: _.in(companionNames), status: 'approved' })
  .field({ openid: true, name: true })
  .get()

// 3. 批量更新同行人的 isOut 状态
if (matchedUsers.data.length > 0) {
  const batch = db.batchUpdate()
  matchedUsers.data.forEach(user => {
    batch.update(usersCollection.doc(user._id), { data: { isOut: true, updatedAt: now } })
  })
  await batch.commit()
}

// 4. 返回匹配结果供前端显示
```

#### handleReturn 修改

```javascript
// 仅更新当前用户状态，不影响同行人
await usersCollection.where({ openid }).update({
  data: { isOut: false, updatedAt: now }
})
```

### 3. 前端修改

#### trip-report.js 提交逻辑

- 提交成功后显示匹配结果提示
- 例如："同行人 张三、李四 已设置为外出状态"

#### trip-report.wxml 提示文案

- 同行人输入框添加提示："多个同行人请用空格分隔，系统将自动识别已注册用户"

### 4. 数据流

```
用户输入同行人 → 前端提交 → 云函数解析姓名 → 查询匹配用户 → 更新isOut状态 → 返回匹配结果
```

## 实现注意事项

1. **姓名匹配精确性**：使用精确匹配，避免误设置
2. **批量操作性能**：使用 `batchUpdate` 批量更新用户状态
3. **返回报备独立性**：明确只更新当前用户，不通知同行人
4. **错误处理**：同行人识别失败不影响主流程，静默处理

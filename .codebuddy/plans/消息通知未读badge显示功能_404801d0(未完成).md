## 产品概述

为个人中心页面的"消息通知"菜单项添加未读消息数量角标（badge），显示格式为"X条未读"。

## 核心功能

- 在云函数中添加获取未读消息数量的接口
- 在 app.js 中封装全局获取未读数量的方法
- 在 profile 页面加载时自动获取并显示未读数量
- 未读数量为0时不显示 badge
- 进入消息中心后刷新 badge 状态

## 技术栈

- 云函数: Node.js (wx-server-sdk)
- 前端: 微信小程序原生框架
- 数据库: CloudBase NoSQL (notifications 集合)

## 实现方案

### 1. 云函数扩展

在 `cloudfunctions/notificationManager/index.js` 中新增 `getUnreadCount` action：

- 查询当前用户 `read: false` 的通知数量
- 返回 `{ success: true, data: { count: number } }`

### 2. 全局方法封装

在 `miniprogram/app.js` 中添加 `getUnreadNotificationCount(callback)` 方法：

- 调用 notificationManager 云函数
- 返回未读数量，失败时返回 0

### 3. 页面逻辑实现

修改 `miniprogram/pages/office/profile/profile.js`：

- 在 `syncUserProfile()` 成功后获取未读数量
- 动态更新 `menuGroups[0].items[0].badge` 字段
- 未读数量大于0时显示 "X条未读"，为0时不显示

### 数据流

```
profile.onShow() 
  → syncUserProfile() 
  → app.getUnreadNotificationCount() 
  → notificationManager 云函数 
  → 更新 menuGroups badge
```

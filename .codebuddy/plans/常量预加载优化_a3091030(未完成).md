## 需求概述

在用户登录时预加载系统常量到 app.globalData，实现全局缓存，避免各页面重复调用云函数获取常量。

## 核心功能

1. 在 app.js 的 globalData 中添加常量缓存字段（含时间戳用于过期判断）
2. 在 app.js 添加常量预加载和管理方法（类似现有的用户信息缓存机制）
3. 在 login.js 登录成功后调用常量预加载
4. 修改 constants.js，优先从 app.globalData 获取常量，缓存过期时间设为24小时

## 预期效果

- 用户登录成功后自动加载常量到全局缓存
- 各页面使用常量时可直接从缓存获取，无需重复调用云函数
- 缓存24小时后自动失效，下次获取时会重新加载

## 技术方案

### 实现策略

复用现有的用户信息缓存模式（USER_CACHE_EXPIRE），在 app.js 中实现常量的全局缓存管理。constants.js 模块优先从 app.globalData 读取，未命中或过期时再调用云函数。

### 数据结构设计

```javascript
// app.js globalData 新增字段
globalData: {
  // ... 现有字段
  constantsCache: null,        // 常量缓存数据
  constantsCacheTime: null     // 缓存加载时间戳
}
```

### 缓存流程

```
页面调用 getConstant/getAllConstants
       ↓
检查 app.globalData.constantsCache 是否存在且未过期
       ↓
命中 → 直接返回缓存数据
       ↓
未命中 → 调用云函数获取 → 存入 app.globalData → 返回数据
```

### 关键实现点

1. **缓存过期时间**：24小时（86400000ms）
2. **预加载时机**：login.js 中 handleWxLogin 成功后，跳转首页前
3. **降级策略**：云函数失败时使用默认常量（已有机制）

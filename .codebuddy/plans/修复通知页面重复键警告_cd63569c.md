## 用户需求

修复消息中心点击推送消息时的重复键警告问题。

## 问题根源

`notifications.js` 的 `loadData` 方法中存在多次 `setData` 操作，导致数据状态不一致：

1. 第53-58行：更新 `notifications`
2. 第61-63行：再次设置 `list = notifications`
3. `pagination.js` 的 `loadListData` 又第三次更新 `list`

这三次 `setData` 导致 `wx:key` 出现重复键警告。

## 修复方案

修改 `notifications.js` 的 `loadData` 方法：

- 只更新 `notifications` 特有的状态（notifications 和 hasUnreadNotifications）
- 不再更新 `list`（完全由 `pagination.js` 管理）
- 返回格式化后的数据给 `pagination.js` 统一处理

## 修改文件

- `miniprogram/pages/office/notifications/notifications.js` - 修改 `loadData` 方法，移除对 `list` 的重复更新

## 修改内容

将第44-76行的 `loadData` 方法修改为：

```javascript
loadData(params) {
  return new Promise((resolve, reject) => {
    app.getNotifications(params, (result) => {
      if (result && result.data) {
        const formattedData = result.data.map(n => ({
          ...n,
          timeText: formatTime(n.createdAt)
        }))

        const hasUnread = formattedData.some(n => !n.read)
        
        // 只更新 notifications 特有的状态，不更新 list
        // list 由 pagination.js 统一管理
        if (this.data.page === 1) {
          this.setData({
            notifications: formattedData,
            hasUnreadNotifications: hasUnread
          })
        } else {
          this.setData({
            notifications: [...this.data.notifications, ...formattedData],
          })
        }

        resolve({
          data: formattedData,
          hasMore: result.hasMore
        })
      } else {
        resolve({
          data: [],
          hasMore: false
        })
      }
    })
  })
},
```

## 关键改动

1. 删除第61-63行的重复 `setData`（`list: this.data.notifications`）
2. 将 `notifications` 的更新逻辑合并到条件判断中
3. 保持返回值格式不变，让 `pagination.js` 统一管理 `list`
